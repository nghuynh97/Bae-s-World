**Props interface:**
```typescript
interface RoutineProductPickerProps {
  routineId: string;
  currentStepProductIds: string[];  // IDs of products already in the routine
  allProducts: ProductData[];       // Full product list from page-level fetch
  categories: { id: string; name: string; slug: string }[];
  onStepAdded: () => void;          // Callback to trigger router.refresh() in parent
}
```
Where `ProductData` matches the shape from `product-grid.tsx` (import or re-declare the type).

**Component structure (top to bottom inside the Dialog):**

1. **Trigger button**: An "Add step" outline button with Plus icon (lucide `Plus`, 16px, left of text). Placement: `mt-4`, left-aligned, auto-width. Use `DialogTrigger` wrapping a `<Button variant="outline">`.

2. **DialogContent**: Override default `sm:max-w-sm` with `sm:max-w-md`. Add `max-h-[80vh] overflow-hidden flex flex-col` to constrain height and enable internal scrolling.

3. **DialogHeader + DialogTitle**: Title text is `"Add step"` with `font-bold` class override on DialogTitle.

4. **Search input**: Use the project `Input` component from `@/components/ui/input`. Wrap in a `relative` div. Add `Search` icon (lucide) positioned `absolute top-1/2 left-3 h-4 w-4 -translate-y-1/2 text-muted-foreground`. Input has `pl-9` and `placeholder="Search products..."`. Wire to `searchQuery` state (`useState('')`). Filter is instant (no debounce needed — client-side).

5. **Category filter pills**: Render a simplified inline category filter (NOT the full `BeautyCategoryFilter` component — it hardcodes "Favorites" which we do not want in the picker). Instead, build a simple pill row:
   - "All" pill + one pill per category from `categories` prop
   - Active pill: `bg-accent text-white`
   - Inactive pill: `border border-border text-text-secondary hover:border-accent hover:text-accent`
   - Pill classes: `rounded-full px-4 py-2 font-body text-sm whitespace-nowrap transition-colors`
   - Wrap in `div className="scrollbar-hide flex gap-2 overflow-x-auto pb-2"`
   - Wire to `activeCategory` state (`useState('all')`)

6. **Scrollable product grid**: Wrap in `div className="flex-1 overflow-y-auto"` (this div scrolls while search+pills stay fixed). Inside, render a `grid grid-cols-3 gap-2` of product thumbnails.

   For each product:
   - Determine if already added: `const isAdded = addedProductIds.has(product.id)` where `addedProductIds` is a Set combining `currentStepProductIds` and `localAddedIds` state.
   - Select image variant: `product.variants.find(v => v.variantName === 'medium') ?? product.variants.find(v => v.variantName === 'small') ?? product.variants[0]`
   - Render an `aspect-square` container with `relative overflow-hidden rounded-md cursor-pointer`
   - If `isAdded`: add `opacity-40 pointer-events-none` to the container. Render a centered `Check` icon overlay: `<div className="absolute inset-0 flex items-center justify-center"><Check className="h-6 w-6 text-white drop-shadow" /></div>`
   - If not added: add `hover:scale-105 transition-transform` for hover feedback. On click, call `handleAddProduct(product.id)`.
   - Image: Use `next/image` with `fill`, `className="object-cover"`, `sizes="120px"`, `alt={product.name}`.
   - Below image (inside the aspect-square or just below): product name as `<p className="mt-1 truncate text-xs text-text-secondary">{product.name}</p>` — but only if this fits the grid aesthetic. Per UI-SPEC, product name is 12px/400 weight truncated single line. Place it OUTSIDE the aspect-square div but inside the grid cell wrapper.

7. **Filtering logic** (combine search + category):
   ```typescript
   const filteredProducts = allProducts.filter((p) => {
     const matchesSearch = !searchQuery ||
       p.name.toLowerCase().includes(searchQuery.toLowerCase()) ||
       (p.brand && p.brand.toLowerCase().includes(searchQuery.toLowerCase()));
     const matchesCategory = activeCategory === 'all' || p.categorySlug === activeCategory;
     return matchesSearch && matchesCategory;
   });
   ```

8. **Empty state**: When `filteredProducts.length === 0`, show:
   `<p className="py-12 text-center text-sm text-text-secondary">No products match your search</p>`

9. **Add handler** (optimistic, dialog stays open):
   ```typescript
   const [localAddedIds, setLocalAddedIds] = useState<Set<string>>(new Set());
   const addedProductIds = new Set([...currentStepProductIds, ...localAddedIds]);

   const handleAddProduct = async (productId: string) => {
     setLocalAddedIds(prev => new Set([...prev, productId]));
     try {
       await addRoutineStep(routineId, productId);
       onStepAdded();
     } catch {
       setLocalAddedIds(prev => {
         const next = new Set(prev);
         next.delete(productId);
         return next;
       });
       toast.error('Failed to add step \u2014 please try again');
     }
   };
   ```

10. **Dialog state management**: Use `<Dialog open={open} onOpenChange={setOpen}>` pattern. When dialog closes (`onOpenChange(false)`), reset `searchQuery` to `''`, reset `activeCategory` to `'all'`, and clear `localAddedIds`. This ensures a fresh state on re-open.
