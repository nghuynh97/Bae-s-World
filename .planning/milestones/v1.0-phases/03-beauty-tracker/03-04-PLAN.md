1. Create `src/components/beauty/beauty-category-manager.tsx`:

"use client" component. Props: `open: boolean`, `onOpenChange: (open: boolean) => void`, `onCategoriesChanged: () => void` (callback to refresh parent's category list).

Replicate the exact pattern from `src/app/(private)/admin/categories/page.tsx` but adapted as a Dialog instead of a full page:

```typescript
'use client';

import { useState, useEffect, useRef } from 'react';
import { Trash2, Settings2 } from 'lucide-react';
import { toast } from 'sonner';
import {
  getBeautyCategories,
  createBeautyCategory,
  updateBeautyCategory,
  deleteBeautyCategory,
} from '@/actions/beauty-categories';
import { Input } from '@/components/ui/input';
import { Button } from '@/components/ui/button';
import {
  Dialog,
  DialogContent,
  DialogHeader,
  DialogTitle,
  DialogDescription,
  DialogFooter,
  DialogClose,
} from '@/components/ui/dialog';
```

State: categories array, newName string, editingId, editName, deleteId, isAdding, isDeleting (same as portfolio pattern).

- Load categories via getBeautyCategories() on mount and when dialog opens.
- Category list: each row shows displayOrder number, category name (click to inline edit), "(default)" label for isDefault===1 items, Trash2 delete button.
- Inline edit: clicking name sets editingId and shows Input. Enter or blur saves via updateBeautyCategory(). Escape cancels.
- Add new: Input + "Add Category" button at bottom. Enter key also triggers add. Calls createBeautyCategory({ name }).
- Delete: Trash2 click sets deleteId, opens nested confirmation Dialog. Confirmation calls deleteBeautyCategory(). Error messages from server ("Cannot delete default categories", "Cannot delete category with existing products") shown via toast.error.
- After any successful add/edit/delete: reload categories via getBeautyCategories() AND call onCategoriesChanged() to refresh the parent filter pills.
- Toast messages: "Category created", "Category updated", "Category deleted" on success.

Wrap all content in the outer Dialog controlled by `open`/`onOpenChange` props:

```tsx
<Dialog open={open} onOpenChange={onOpenChange}>
  <DialogContent className="max-w-md">
    <DialogHeader>
      <DialogTitle>Edit Categories</DialogTitle>
    </DialogHeader>
    {/* Category list, add form, delete confirmation */}
  </DialogContent>
</Dialog>
```
