## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Grid gap between product thumbnails, compact element spacing |
| md | 16px | Dialog body padding, section padding inside dialog |
| lg | 24px | Routine card padding (existing p-6) |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions: Touch targets for product grid items use the full aspect-square cell area (minimum 100px on mobile) which does not align to the 8-point scale but is dictated by the 3-column grid in a constrained dialog width.

## Interaction Contract

### Picker Dialog Flow

1. **Trigger**: User taps "Add step" button (outline variant) below the routine steps list
2. **Dialog opens**: fade-in + zoom-in animation (existing DialogContent `data-open:animate-in`)
3. **Layout top-to-bottom**:
   - DialogHeader with DialogTitle "Add step"
   - Search input with Search icon (left-aligned, `pl-9`)
   - Category filter pills (horizontal scroll, same as Products tab but without "Favorites")
   - Scrollable product photo grid (3-column, `gap-2`, `aspect-square` cells)
4. **Filtering**: Typing in search filters the grid client-side by product name or brand (case-insensitive `includes()`)
5. **Category filtering**: Tapping a category pill filters the grid to that category (client-side, same pattern as ProductGrid)
6. **Already-added products**: Cards show `opacity-40` + centered white Check icon overlay + `pointer-events-none`
7. **Adding a product**: Tap an available (not dimmed) product card -> optimistic add to local `addedIds` Set -> call `addRoutineStep` server action -> call `onStepAdded` callback -> product immediately gets dimmed+checkmark treatment
8. **Multi-add**: Dialog stays open after adding. User can add multiple products in one session.
9. **Error handling**: If `addRoutineStep` fails, revert optimistic state, show `toast.error("Failed to add step")`
10. **Close**: User taps X button or clicks backdrop. Dialog closes with fade-out + zoom-out animation. Routine card shows new steps.

### Dialog Sizing

| Breakpoint | Width | Height |
|------------|-------|--------|
| Mobile (<640px) | `calc(100% - 2rem)` (default DialogContent) | `max-h-[80vh]` |
| Desktop (>=640px) | `sm:max-w-md` (448px) | `max-h-[80vh]` |

The search bar and category pills are sticky at the top. The grid area below scrolls independently with `overflow-y-auto`.

| Element | Copy |
|---------|------|
| Primary CTA | "Add step" (button label on trigger) |
| Dialog title | "Add step" |

## States Matrix

| State | Visual Treatment |
|-------|-----------------|
| Dialog closed | "Add step" outline button visible below steps |
| Dialog open, no filter | All products visible in grid, already-added dimmed |
| Dialog open, search active | Grid filtered to matching products |
| Dialog open, category selected | Grid filtered to selected category |
| Dialog open, search + category | Both filters combined (AND logic) |
| Product available | Normal opacity, cursor-pointer, hover:scale-105 transition |
| Product already added | opacity-40, pointer-events-none, centered Check icon overlay |
| Product being added | (optimistic) immediately transitions to already-added state |
| Add failed | Reverts to available state, toast.error shown |
| No search results | "No products match your search" centered text |
| No products at all | Grid area empty (edge case — unlikely for active user) |
