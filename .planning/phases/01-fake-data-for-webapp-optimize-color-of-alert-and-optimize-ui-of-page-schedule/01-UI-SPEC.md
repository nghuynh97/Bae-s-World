### Phase-Specific Colors: Toast Backgrounds

These are NEW color applications for typed toast notifications. Colors are soft pastels that complement the rose gold accent (#e8b4b8) and lavender dominant (#f8f6ff).

| Toast Type | Background | Border | Text | Icon Color |
|------------|------------|--------|------|------------|
| Success | #ecfdf5 (`emerald-50`) | #a7f3d0 (`emerald-200`) | #065f46 (`emerald-800`) | #065f46 (`emerald-800`) |
| Error | #fef2f2 (`red-50`) | #fecaca (`red-200`) | #991b1b (`red-800`) | #991b1b (`red-800`) |
| Warning | #fffbeb (`amber-50`) | #fde68a (`amber-200`) | #92400e (`amber-800`) | #92400e (`amber-800`) |
| Default | var(--popover) | var(--border) | var(--popover-foreground) | inherited |

Implementation note: Use Tailwind `!important` modifiers (`!bg-emerald-50`) to override Sonner inline styles, applied via `toastOptions.classNames.success/error/warning`. If `!important` fails to override, fall back to CSS custom properties on `[data-sonner-toast][data-type="success"]` selectors in `globals.css`.

## Interaction Patterns

### Toast Notifications

| Interaction | Behavior |
|-------------|----------|
| Appearance | Slide in from top-center (existing Sonner behavior) |
| Duration | 3000ms auto-dismiss (existing) |
| Dismiss | Swipe or click X (existing Sonner behavior) |
| Visual change | Background color now reflects toast type (green/red/amber) instead of uniform white |

### Admin Seed Button

| Interaction | Behavior |
|-------------|----------|
| Visibility | Only rendered when current user is boyfriend (invite_codes check) |
| Click | Opens confirmation dialog |
| Confirm | Button shows "Seeding..." with spinner, disables interaction |
| Complete | Success toast, page data refreshes via `revalidatePath('/')` |
| Error | Error toast with retry guidance |

### Stat Card Grid (Schedule Page)

| Viewport | Layout |
|----------|--------|
| Mobile (<640px) | Horizontal scroll with snap, 260px min-width per card |
| Desktop (>=640px) | 3-column grid (`sm:grid-cols-3`) |
