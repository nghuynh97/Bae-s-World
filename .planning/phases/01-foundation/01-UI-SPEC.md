# Phase 1 — UI Design Contract

> Visual and interaction contract for the Foundation phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn/ui (CLI v4) |
| Preset | To be initialized with `npx shadcn@latest init` after project scaffolding |
| Component library | Radix UI (via shadcn) |
| Icon library | Lucide React (ships with shadcn) |
| Heading font | Playfair Display (Google Fonts, via `next/font/google`) |
| Body font | Inter (Google Fonts, via `next/font/google`) |
| CSS framework | Tailwind CSS v4 with `@theme` directive for design tokens |

### Font Rationale

- **Playfair Display** for headings: editorial, fashion-magazine serif as specified in user decisions. High contrast letterforms create the elegant, premium feel.
- **Inter** for body: highly readable sans-serif with excellent screen rendering at small sizes. Its neutral character complements Playfair Display without competing. Optimized for UI text with tabular numbers and contextual alternates.

## Spacing Scale

Declared values (all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline element padding, badge padding |
| sm | 8px | Compact element spacing, input padding-y, gap between label and input |
| md | 16px | Default element spacing, card padding, button padding-x |
| lg | 24px | Section padding, form field vertical gaps, card content padding |
| xl | 32px | Layout gaps between major sections, nav bar padding |
| 2xl | 48px | Major section vertical breaks, page section margins |
| 3xl | 64px | Page-level top/bottom padding, hero section spacing |

Exceptions:
- Mobile bottom tab bar height: 56px (14 x 4, accommodates icons + labels with comfortable touch targets)
- Desktop top nav height: 64px (matches 3xl token)
- Touch targets: minimum 44px (11 x 4, per WCAG 2.5.5)

## Typography

| Role | Size | Weight | Line Height | Font |
|------|------|--------|-------------|------|
| Body | 16px | 400 (regular) | 1.5 | Inter |
| Label | 14px | 500 (medium) | 1.4 | Inter |
| Heading | 24px | 700 (bold) | 1.2 | Playfair Display |
| Display | 36px | 700 (bold) | 1.1 | Playfair Display |

### Additional Typography Rules

- "Funnghy's World" logo text: Display size (36px) in Playfair Display 700, colored with accent (#E8B4B8)
- Form input text: Body size (16px) Inter 400 (prevents iOS zoom on focus)
- Button text: Label size (14px) Inter 500, uppercase tracking at 0.025em
- Caption/helper text: 12px Inter 400, line-height 1.4, used for form hints and metadata timestamps
- Maximum content width: 1280px (80rem)
- Paragraph max-width: 640px (40rem) for comfortable reading

### Accent Reserved For

The accent color #E8B4B8 is reserved exclusively for:
1. "Funnghy's World" logo text
2. Primary CTA buttons (background)
3. Active/selected navigation indicator (bottom tab bar dot, top nav underline)
4. Focused input border color
5. Upload dropzone active border (when dragging over)
6. Progress bar fill during upload

Accent is NOT used for: card backgrounds, section backgrounds, all links, or decorative borders.

| shadow-lg | 0 8px 24px rgba(232, 180, 184, 0.16) | Modals, dropdowns, popover |

## Border Radius

| Token | Value | Usage |
|-------|-------|-------|
| radius-sm | 6px | Badges, small chips, tags |
| radius-md | 10px | Buttons, inputs, small cards |
| radius-lg | 16px | Cards, modals, image containers |

Rationale: "Gently rounded" as specified -- not pill-shaped (which would be full radius), not sharp (0). The 10px default for interactive elements feels soft without being juvenile.

## Layout Specifications

### Desktop Top Nav (viewport >= 768px)

- Height: 64px
- Background: #FFFFFF with shadow-sm bottom
- Left: LogoText component
- Right: nav links (Portfolio, Beauty, Journal, About) + user dropdown
- Nav link style: Inter 14px 500, text-secondary color, accent underline on active
- Max content width: 1280px, centered
- Public visitors see: Portfolio, About, Login button
- Authenticated users see: Portfolio, Beauty, Journal, About, user dropdown

### Mobile Bottom Tab Bar (viewport < 768px)

- Height: 56px
- Fixed to bottom of viewport
- Background: #FFFFFF with shadow-lg top (inverted, shadow goes upward)
- 4 tabs for authenticated: Portfolio, Beauty, Journal, Profile
- 3 tabs for public: Portfolio, About, Login
- Each tab: Lucide icon (24px) above label (10px Inter 500)
- Active tab: accent color icon + label, inactive: text-secondary
- Active indicator: 4px diameter dot in accent below the icon

### Login Page

- Full viewport height, centered content
- Background: dominant (#F8F6FF)
- LogoText at top, Display size (36px)
- Subtitle: "Welcome back" in text-secondary, Body size
- Form width: 360px max, centered
- Fields: Email, Password
- Submit button: full-width, accent background, white text, radius-md
- Below form: "First time? Enter your invite code" link in text-secondary
- Invite code variant: same layout, single code field, then transitions to password creation

### Invite Code Setup Flow

- Step 1: Enter 6-character invite code (alphanumeric, auto-uppercase)
- Step 2: If valid, show email + create password + confirm password fields
- Step 3: Submit creates account and logs in
- Invite code format: 6 uppercase alphanumeric characters (e.g., "FNGH01", "BFRD02")
- Invalid code: inline error below input, input border turns destructive color

### Dashboard (Post-Login Home)

- Page heading: "Welcome back, [Name]" in Heading size (24px) Playfair Display
- Layout: 2-column grid on desktop (gap: xl/32px), stacked on mobile (gap: lg/24px)
- 3 quick-access cards: Portfolio, Beauty Tracker, Photo Journal
- Each card: radius-lg, shadow-md, white background
- Card content: preview thumbnail (if recent content exists), section title in Label 500, recent activity subtitle in caption (12px), accent-colored arrow icon on right
- Empty card state: section title + descriptive subtitle + accent outline dashed border where thumbnail would be

### Image Upload Zone

- Container: dashed 2px border in border-default, radius-lg, padding 2xl (48px)
- Center: Lucide Upload icon (48px) in text-secondary
- Below icon: "Drag photos here or click to browse" in Body 400 text-secondary
- Below text: "JPEG, PNG, WebP accepted" in caption 12px text-secondary
- Active drag state: border becomes accent solid, background becomes rgba(232, 180, 184, 0.05)
- Per-file progress: file name (Label 14px 500) + progress bar (accent fill, 4px height, radius-full) + percentage (caption 12px)
- Upload complete: green success checkmark replaces progress bar, file name stays

## State Specifications

### Loading States

- **Page load:** Full skeleton matching page layout. Cards show skeleton rectangles with radius-lg and subtle pulse animation (opacity 0.5 to 1, 1.5s duration, ease-in-out).
- **Nav loading:** Logo text renders immediately (static), nav links show skeleton bars (80px wide, 12px tall).
- **Dashboard cards:** Each card shows skeleton thumbnail (aspect-ratio 16/9), skeleton title bar (60% width), skeleton subtitle bar (40% width).
- **Upload in progress:** Progress bar animates width. File name and percentage visible. No spinner -- the progress bar is the indicator.

### Error States

- **Form validation errors:** Red border (#DC2626) on the invalid input, error message in 12px Inter 400 #DC2626 below the input, appears immediately on blur or submit.
- **Toast errors:** Sonner toast, destructive variant, appears top-right on desktop, top-center on mobile. Auto-dismiss after 5 seconds. Contains error icon + message.
- **Network/server errors:** Full-width inline alert at top of the form or content area. Light red background (rgba(220, 38, 38, 0.05)), red left border (4px), destructive text.

### Empty States

- **Dashboard with no content:** Each card shows its title and subtitle, with a dashed accent border outline where the thumbnail would be. Body text below: "Nothing here yet."
- **Upload zone idle:** Dashed border, upload icon, instructional text (as specified in Layout Specifications).

## Interaction Patterns

### Login Form

- Email field: validates on blur (basic email format)
- Password field: validates on submit (minimum 8 characters)
- Submit button: shows loading state (text changes to "Signing in..." + disabled) during auth request
- On success: redirect to dashboard
- On failure: shake animation on form (subtle, 300ms), error message appears

### Invite Code Flow

- Auto-uppercase as user types
- Auto-advance to account creation when 6 characters entered and code validates
- Invalid code: inline error, field clears and refocuses

### Image Upload

- Drag events: dragenter shows active state, dragleave reverts, drop triggers upload
- Click anywhere in zone opens file picker
- Multiple file selection supported
- Each file gets its own progress bar, stacked vertically
- Files upload in parallel (max 3 concurrent)
- Completed files show green check, failed files show red X with "Retry" link
- After all files complete: success toast with total count

### Navigation

- Desktop nav links: underline slides in from left on hover (200ms ease), accent color underline on active page
- Mobile tab bar: instant highlight on tap, no transition delay (feels responsive like Instagram)
- Page transitions: reserved for Phase 5 (no transition animations in Phase 1)

## Accessibility Requirements

- All form inputs have associated labels (via `<label htmlFor>`)
- Focus ring: 2px accent color outline with 2px offset on all interactive elements
- Color contrast: text-primary (#2D2235) on dominant (#F8F6FF) = ratio ~12:1 (passes AAA)
- Color contrast: text-secondary (#6B5F76) on dominant (#F8F6FF) = ratio ~4.8:1 (passes AA)
- Accent (#E8B4B8) as button background with white text: ratio ~3.2:1 -- use text-primary (#2D2235) as button text instead for AA compliance
- Upload zone is keyboard-accessible (Enter/Space triggers file picker)
- Mobile bottom tab bar items have aria-label attributes
- Login form error messages use aria-live="polite" for screen reader announcement

### Accent Button Accessibility Correction

Due to contrast requirements, accent-background buttons (#E8B4B8) must use dark text (#2D2235) instead of white. This maintains the soft, feminine aesthetic while passing WCAG AA.
